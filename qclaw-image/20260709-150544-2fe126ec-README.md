# BI Daily Report Migration Pack

这份迁移包只描述 `2026-07-09` 仍在使用的实现。

不包含：
- 旧仓库路径 `product-codex-platform`
- 已废弃的 `run-bi-daily-report-launchd.sh`
- GitHub Actions 作为 active 调度入口
- 任何旧 OpenClaw/旧 cron 方案

## 1. 当前活跃实现

当前日报的真实入口是：

```bash
bash 30-scripts/reports/bi-daily-report.sh
```

这层入口做的事情很少：
- 自动推导仓库根目录
- 设置 `REPORT_DIR`
- 设置 `SHOTS_DIR`
- 设置 `DINGTALK_CONFIG`
- 调用 `30-scripts/reports/bi-daily-report-cloud.sh`

真正的执行主体是：

```bash
30-scripts/reports/bi-daily-report-cloud.sh
```

## 2. 当前内容结构

日报当前包含 5 段：

1. 充值金额
2. 注册首日充值>200元付费用户占比
3. 场景弹窗触发及充值情况
4. cp爬楼情况
5. 推币挑战

其中：
- BI1 会同时生成 `充值金额` 和 `新用户数据`
- BI4 当前使用 `2100` 宽视口和对应裁切参数，这是最新修正后的版本

## 3. 目录说明

迁移包里的文件分为两类：

- `files/30-scripts/reports/`
  - 当前活跃的日报脚本和发送辅助脚本
- `files/00-context/local/ddingtalk.example.json`
  - 脱敏后的钉钉配置模板

## 4. 依赖

目标机器至少需要这些依赖：

- macOS
- `bash`
- `python3`
- Python `Pillow`
- `curl`
- `agent-browser`

说明：
- 这些脚本依赖 `agent-browser` 打开 BI 页面、滚动、截图、执行页面内 JS。
- 没有 `agent-browser`，这套脚本不能正常工作。

## 5. 钉钉配置

默认会从下面任一位置读取配置：

1. 环境变量 `DINGTALK_CONFIG`
2. 仓库内 `00-context/local/ddingtalk.json`
3. 用户目录 `~/.openclaw/secrets/ddingtalk.json`

推荐做法：

1. 复制模板文件：

```bash
cp 00-context/local/ddingtalk.example.json 00-context/local/ddingtalk.json
```

2. 填入实际值：
- `clientId`
- `clientSecret`
- `webhook`
- 如果机器人开启签名，再补 `webhookSecret`

发送优先级：
- 优先走 `webhook`
- 如果没有 `webhook`，才退回到 `DD_KEY/DD_SECRET + 单聊发送`

## 6. 当前调度方式

这台机器上当前活跃调度不是 launchd 包装脚本，而是两层：

1. macOS 电源唤醒：

```bash
pmset repeat wakepoweron MTWRFSU 08:58:00
```

2. Codex 本地自动化：

- 名称：`运营日报 9点`
- 执行目录：仓库根目录
- 命令：`bash 30-scripts/reports/bi-daily-report.sh`

也就是说，迁移时如果要完整照搬，应该复制的是：
- 日报脚本目录
- 钉钉配置
- 调度策略本身

而不是去找旧的 launchd 包装脚本。

## 7. 已知运行特性

### 7.1 唤醒问题

这套方案依赖 GUI 会话里的 `Codex.app` 和 `agent-browser` 可用。

实践里要注意：
- `pmset` 唤醒成功，不代表 GUI 应用一定恢复
- 如果 `Codex.app` 没有恢复，9 点自动化可能不会按时跑
- 所以目标机器上应重点验证：
  - `8:58` 唤醒后，`Codex.app` 是否仍处于运行态
  - `agent-browser` 是否可正常截图

### 7.2 输出目录

默认输出目录：

```bash
01-memory/artifacts/bi/reports
```

主要产物包括：
- `充值金额.png`
- `新用户数据.png`
- `注册首日充值大户.png`
- `场景弹窗.png`
- `cp爬楼情况.png`
- `推币挑战.png`
- `bi1-raw.json`

## 8. 迁移步骤

在新电脑上建议按这个顺序：

1. 准备仓库目录结构
2. 放入 `files/30-scripts/reports/`
3. 放入 `files/00-context/local/ddingtalk.example.json`
4. 改名为 `ddingtalk.json` 并填好凭证
5. 安装并验证：
   - `python3`
   - `Pillow`
   - `curl`
   - `agent-browser`
6. 手动跑一次：

```bash
bash 30-scripts/reports/bi-daily-report.sh
```

7. 确认 5 段全部完成，最后出现：

```bash
===== 报告发送完成 =====
```

8. 再配置调度
